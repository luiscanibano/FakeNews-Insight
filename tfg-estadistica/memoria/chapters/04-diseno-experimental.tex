\chapter{Diseño experimental}\label{cap:diseno}

\section{Dataset FEVER}

\subsection{Composición y particiones}

El dataset FEVER \citep{thorne2018fever} contiene \emph{claims}
sintéticos generados por anotadores humanos a partir de artículos de
Wikipedia. Cada \emph{claim} se etiqueta como \textsc{supported}
(\textsc{sup}), \textsc{refuted} (\textsc{ref}) o
	extsc{not-enough-info} (\textsc{nei}). Para evitar depender del
volcado completo de Wikipedia, los experimentos emplean la variante
FEVER-NLI publicada como \texttt{pietrolesci/nli\_fever}, que ya
proporciona pares \emph{premise}--\emph{hypothesis} con evidencia
textual.

La distribución descargada contiene particiones \emph{train},
\emph{dev} y \emph{test}. La partición \emph{test} pública mantiene la
etiqueta ciega \emph{not available}, por lo que no puede usarse para
evaluación supervisada. En consecuencia, se descarta esa partición y se
divide \emph{dev} en validación y prueba mediante un corte 50/50
estratificado con semilla fija (\texttt{random\_state=42}). Tras el
preprocesado quedan 196\,805 ejemplos de entrenamiento, 9\,826 de
validación y 9\,826 de prueba.

\subsection{Preprocesado}

Para los experimentos se construye una tabla
$(\text{claim}, \text{evidencia}, \text{etiqueta})$ donde cada fila
corresponde a un par claim--evidencia. Las etiquetas se normalizan a
las tres clases anteriores y se eliminan las filas sin etiqueta
supervisada. La implementación del preprocesado se encuentra en
\path{research-stats/research_stats/data/preprocess.py}.

\section{Métricas de evaluación}

Sea $C \in \R^{K \times K}$ la matriz de confusión con
$C_{ij}$ el número de ejemplos de clase verdadera $i$ predichos como
$j$. Definimos para cada clase $k$:
\[
  \mathrm{prec}_k = \frac{C_{kk}}{\sum_j C_{jk}}, \quad
  \mathrm{rec}_k = \frac{C_{kk}}{\sum_j C_{kj}}, \quad
  F1_k = \frac{2\,\mathrm{prec}_k \mathrm{rec}_k}{\mathrm{prec}_k + \mathrm{rec}_k}.
\]
La \emph{exactitud} (\emph{accuracy}) es
$\mathrm{acc} = N^{-1} \sum_k C_{kk}$ y el F1 \emph{macro} es
$F1_{\text{macro}} = K^{-1} \sum_k F1_k$.

Adicionalmente reportamos:
\begin{itemize}[leftmargin=*,itemsep=0.2em]
  \item \emph{Log-loss} multiclase (NLL):
        $\mathrm{NLL} = -N^{-1} \sum_i \log \hat{p}_{y_i}(x_i)$.
  \item \emph{Brier score} multiclase:
        $\mathrm{BS} = N^{-1} \sum_i \sum_k
        \bigl(\hat{p}_k(x_i) - \mathbb{1}\{y_i = k\}\bigr)^2$.
  \item \emph{Expected Calibration Error} (ECE) con $M = 15$ bins
        equidistantes en $[0, 1]$.
\end{itemize}

\section{Protocolo de evaluación}

El experimento final se ejecuta con semilla fija \texttt{42}. Para la
comparación entre el baseline y el modelo neuronal se aplica el test
de McNemar pareado (capítulo~\ref{cap:tests}) sobre la partición de
prueba; para la diferencia de F1 macro se construye un intervalo de
confianza por \emph{bootstrap} no paramétrico con $B = 1\,000$
remuestreos.

\section{Recursos computacionales}

Los modelos lineales se entrenan en CPU. El \emph{fine-tuning} de
DeBERTa-v3-small se realiza en una NVIDIA GTX 1660 SUPER de 6\,GB con
precisión mixta \texttt{fp16}; por limitación de memoria se usa tamaño
de lote físico 8 y acumulación de gradiente 4, equivalente a lote
efectivo 32.
